  SECTION 6: DEBUGGING STEPS FOR NEW APPS
═══════════════════════════════════════════════════════════════════════════════════

WHEN ADDING A NEW APP:

Step 1: Add app object to 'apps' array
  Format: { id: 'unique-id', name: 'App Name', icon: 'icon-url', file: 'file-path' }
  
Step 2: Verify required properties
  ✓ id: unique identifier (no spaces, no duplicates)
  ✓ name: display name for UI
  ✓ icon: local path (../Other/...) or CDN URL (https://...)
  ✓ file: local path or external URL

Step 3: Check file paths
  For local files: Use relative paths from app location
    From Important_Files/Desktop.html:
      - Local app in Important_Files/: 'AppName.html'
      - Local app in Other/: '../Other/AppName.html'
      - Game in Apps/Games/: '../Apps/Games/GameName.html'
  
  For external URLs:
    - Must include https://
    - Full URL required
    - May have CORS/CSP restrictions

Step 4: Clear browser cache
  Ctrl+Shift+Delete (most browsers)
  or Open DevTools → Application → Clear Storage

Step 5: Check browser console
  F12 or Ctrl+Shift+I
  Look for errors in Console tab
  Check Network tab for 404s

Step 6: Test in browser
  1. Refresh (F5)
  2. Should see app in desktop icons
  3. Should see app in start menu
  4. Click app to launch
  5. Window should open with content

  SECTION 7: KNOWN ISSUES & LIMITATIONS
═══════════════════════════════════════════════════════════════════════════════════

LIMITATION_001: External URLs in iframe
Status: Known Limitation
Issue: Many external websites block iframe embedding via X-Frame-Options
Impact: CoAWK and similar external apps may show "Refused to connect"
Workaround: Use window.open() to open in new tab instead
Expected_Fix: Version 2.1 (implement popup window mode for external sites)

LIMITATION_002: Mobile responsiveness
Status: Not Implemented
Issue: Desktop layout doesn't adapt well to mobile screens
Impact: Icons overflow, interface hard to use on phones/tablets
Workaround: Use desktop/tablet in landscape mode
Expected_Fix: Version 3.0 (planned responsive redesign)

LIMITATION_003: Multi-window performance
Status: Known Issue
Issue: Opening 20+ windows causes performance degradation
Impact: Window dragging becomes choppy, memory usage increases
Workaround: Close unused windows, use taskbar to hide apps
Expected_Fix: Version 2.1 (optimize window rendering)

LIMITATION_004: Custom app storage limit
Status: Known Issue
Issue: localStorage has 5MB limit for custom apps
Impact: Cannot add many large custom applications
Workaround: Use small local files or external URLs
Expected_Fix: Version 3.0 (implement IndexedDB for larger storage)

HOW TO ADD A NEW APP SUCCESSFULLY:

1. Edit Important_Files/Desktop.html

   { id: 'coawk', name: 'CoAWK', icon: 'https://...logo.png', file: 'https://coawk-website.com/' }

EXAMPLE FOR LOCAL APP:
   { id: 'myapp', name: 'My App', icon: '../Other/myapp.png', file: '../Apps/Games/MyApp.html' }

IF APP DOESN'T APPEAR:
  - Check browser console (F12)

  - Clear browser cache
  - Refresh page
